## 📂 FTPでのアップロード手順（FileZilla例）

### 1. 接続情報の設定
- ホスト: `smash-inc.co.jp` または `sv〇〇〇〇.xserver.jp`
- ユーザー名: （Xserverサーバーパネルで確認）
- パスワード: （サーバーパネルのパスワード）
- ポート: 21

### 2. アップロード先
リモート側（右側）で以下のフォルダを開く:
```
/smash-inc.co.jp/public_html/
```

### 3. ファイルをドラッグ＆ドロップ
ローカル側（左側）で以下を選択して、リモート側にドラッグ:
- `index.html`
- `news-detail.html`
- `style.css`
- `assets/` フォルダ（フォルダごと）

### 4. 転送完了を確認
リモート側に以下の構成になっていればOK:
```
/smash-inc.co.jp/public_html/
├── index.html
├── news-detail.html
├── style.css
└── assets/
    ├── hero-keyvisual.png
    ├── member-minakawa.png
    └── （その他のファイル）
```

## 🌐 Xserverファイルマネージャーでのアップロード手順

### 1. ファイルマネージャーを開く
1. Xserverサーバーパネルにログイン
2. 「ファイルマネージャー」をクリック
3. `/smash-inc.co.jp/public_html/` に移動

### 2. ファイルのアップロード

#### HTMLとCSSファイル
1. 「アップロード」ボタンをクリック
2. `index.html`、`news-detail.html`、`style.css` を選択
3. アップロード実行

#### assetsフォルダ
1. 「フォルダ作成」で `assets` フォルダを作成
2. `assets` フォルダに移動
3. 「アップロード」で assets 内のすべてのファイルを選択してアップロード

## ✅ アップロード完了後の確認

### 1. ファイルの確認
ブラウザで直接ファイルにアクセスして確認:
- https://smash-inc.co.jp/index.html （リダイレクトして https://smash-inc.co.jp/ になればOK）
- https://smash-inc.co.jp/style.css （CSSが表示されればOK）
- https://smash-inc.co.jp/assets/smash-logo.png （画像が表示されればOK）

### 2. トップページの表示確認
https://smash-inc.co.jp にアクセスして、デザインが崩れていないか確認

### 3. 注意事項
- この時点ではNewsセクションは正常に動作しません（WordPress未設置のため）
- WordPress設置後に再度確認してください

---

## 🔄 更新時の手順

サイトの内容を更新する場合:

1. ローカルで `index.html` や `style.css` を編集
2. FTPまたはファイルマネージャーで**該当ファイルのみ**上書きアップロード
3. ブラウザでキャッシュをクリアして確認（Ctrl+F5 または Cmd+Shift+R）
